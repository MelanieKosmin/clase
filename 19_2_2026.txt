Git:sistema de control de veriones

CLI: comand line interface

clear: limpia la terinal

cd: change directory
~ :raiz del sistema
ls. lista de directorios y ficheros del directorio actual

Respositorio: es un directorio gestionado por Git

para crear un repositorio vamos a usar el comendo : GIT INIT : en el direcorio q necesitamos crear el repositorio

git status: estado del repositorio
-----------------------------------------------------------

staging: los ficheros estan controlados por Git
commit: crea una version

para añadir ficheros al staging: es git add NOMBRE_FICHERO
git add .    ...> esto es añadir todos los ficheros q no estèn añadidos

para crear una version(commit): git commit -m"bla bla"
-m : es de commit message(msj)

-x : es una flag(bandera)se llama asi para pasar info al sistema

.Ds_store: son ficheros ocultos, 
-----------------------------------------------------------------

macOS: tiene un fichero de gestion de directorio llamado .Ds_store


GIT: VA A TENER LA POSIBILIDAD E IGNORAR FICHEROS DE UNA CARPETA, CON UN: .gitignore

A medida q vamos programando, cada ciert tiepo podemos hacer git add . para agregar 
cambios al STAGING
git commit -m "bla bla" : para crear una nueva version del proyecto


podemos tener la posiilidad de hacer una lista de veriones(commits)

git log

git checkout: permite cambiar "el estado del repositorio"

git checkout HASH_COMMIT: permite mover el HEAD a una version anterior
------------------------------------------------------------------------

branches: ramas (forma en la q esta estucturada git)


git branch : es para hacer una lista de ramas
git brabch NOMBRE_RAMA: es para crear una rama

MERGE: fusionar ramas
1) asegurarme q estoy en la rama estilo
2) ejecutar git merge RAMA_A_FUSIONAR


git push: subir cambio a un repoitorio remoto
git pull: descargar cambios a de un repositorio remoto

para configurar un remositorio remoto:
en el local: git remore add NOMBRE_REPO_REMOTO URL

cuando subimos por primera vez(tenesmos q especificar todo): git push -u REMOTO local
-u: es para crear un "canal de subida" upstream
